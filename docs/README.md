# Documentation

This folder contains documentation committed to the repository for people working on this project.

## What gets committed here?

- **Onboarding guides**: Documents that help new team members get started
- **Architecture decision records**: Documents explaining why certain technical choices were made
- **Maintenance guides**: Documents describing how to keep the project running smoothly
- **Legacy decision records**: Documents about important past choices that still affect the project
- **Manual workflow documents**: Step-by-step guides for processes that aren't automated yet (important to capture early)

## What doesn't go here?

**Consumer-facing documentation** (documentation for end users) should be handled separately. This folder is for internal project documentation only.
